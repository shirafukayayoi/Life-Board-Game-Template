## 1. 進行システムの変更

### 変更前 → 変更後

| 項目 | 変更前 | 変更後 |
|---|---|---|
| 進行方式 | d3サイコロ（1〜3マス） | 固定：1ラウンド = 1ヶ月 = 1イベント |
| 総ラウンド数 | 16（4年×4季節） | **48**（大学1年4月〜4年3月） |
| ターン内の操作 | サイコロ → 選択 | 選択のみ（サイコロUI削除済み） |
| ブランチ構造 | branch 1/2/3（ルート分岐） | **廃止**。全員が同月のイベントを体験 |
| ラウンド進行 | 手動（ホストが進める） | 全員選択完了 → 自動で次ラウンドへ |

### 設計の意図

大学生活は「何年生の何月に何が起きるか」がある程度固定されている（履修登録・試験・学園祭・就活解禁など）。  
時系列に沿った進行にすることで、プレイヤー同士が同じタイミングで違う選択をする面白さが生まれる。

## 2. イベントシステム（3層構造）

```
Layer 1 │ 固定月イベント
        │ その月にしか起きない出来事。試験・学園祭・就活解禁など。
        │ → data/events/main.json（1〜48）
        │
Layer 2 │ 季節・自由枠プール（フラグで絞り込み → プレイヤーが選択）
        │ 春休み（2月・3月）と夏休み（8月）が対象。
        │ プレイヤーのフラグに応じて利用可能なルートが変わる。
        │ → data/events/vacationPool.json（未作成）
        │
Layer 3 │ 個人ランダム枠（Month 15・18・26・40）
        │ フラグ連動 + 不運バッファーで抽選。
        │ 隣のプレイヤーとは違うイベントが来る「個人差」の演出。
        │ → data/events/randomPool.json（未作成）
```

### Layer 3の個人ランダム枠に入るイベント例

| イベント | 条件 | 楽しい/ペナルティ |
|---|---|---|
| 合コンに誘われる | has_partner で内容変化 | どちらも |
| バイト先でトラブル | job_type != null | ペナルティ寄り |
| 先輩に飲みに誘われる | 常時 | 楽しい寄り |
| 急な出費（車・医療など） | 常時 | ペナルティ |
| SNSがバズる | 常時 | 楽しい |
| 友人の結婚式に招待される | 3年以降 | 楽しい |

## 4. 単位システム

### 変更点

| 項目 | 変更前 | 変更後 |
|---|---|---|
| 卒業必要単位 | 110 | **124** |
| チェックポイント | 留年判定（ペナルティあり） | **警告のみ**（ペナルティなし） |
| 留年判定タイミング | 各年末 | **Month 47のみ** |
| 単位マイナス効果 | あり | **全廃（0に変更）** |

### チェックポイント警告ライン（ペナルティなし）

| タイミング | 警告ライン |
|---|---|
| Month 12（1年3月） | 25単位未満 |
| Month 24（2年3月） | 55単位未満 |
| Month 36（3年3月） | 90単位未満 |
| Month 47（4年2月） | **124単位未満 → 留年確定** |

### 単位配分の設計ガイドライン

- 全力で勉強した場合の最大値：約155〜165単位
- 通常プレイ（バランス型）で卒業可能な設計
- 単位★★（試験）：最大10〜12単位 × 8回
- 単位★★★（卒論提出）：最大15単位 × 1回

## 5. キャリアパス分岐

### フラグ

```
career_path: "standard" | "grad_school" | "entrepreneur" | null
```

Month 30（3年9月）で確定。それ以前は null。

### Month 30 の選択肢

```json
[
  {
    "id": "30-A",
    "label": "就活に集中する",
    "description": "王道ルート。どんなステータスでも戦える。",
    "flagEffects": { "career_path": "standard" }
  },
  {
    "id": "30-B",
    "label": "大学院進学を目指す",
    "description": "研究の道へ。📚 知性7以上推奨。低いと院試で痛い目を見る。",
    "flagEffects": { "career_path": "grad_school" }
  },
  {
    "id": "30-C",
    "label": "起業する",
    "description": "自分でゼロから作る道。🚀 行動力7・知性5以上推奨。甘くない。",
    "flagEffects": { "career_path": "entrepreneur" }
  }
]
```

選択肢は全員に表示される（ハードロックなし）。  
ただし低スペックで挑んだ場合、後続イベントで**相応のしっぺ返し**がくる。

### 低スペック時の分岐

**院進ルート（知性不足）：**

| 知性 | Month 38（院試本番）の成功確率 |
|---|---|
| 7以上 | 75% |
| 4〜6 | 45% |
| 3以下 | 0%（全落ちルートのみ） |

全落ち時 → `flagEffects: { career_failed: true }`  
effects: `health -4, money -2, connections -2, work_tolerance -1`

**起業ルート（スペック不足）：**

| 条件 | Month 39（トラブル発生）の結果 |
|---|---|
| 行動力7以上かつ知性5以上 | 軌道に乗る可能性あり |
| それ以外 | 事業失敗（必須） |

失敗時 → `flagEffects: { career_failed: true }`  
effects: `money -5, health -3, connections -1`

### career_failed フラグ

`determineEnding()` にて最優先で判定される（単位が124以上でも発動）。  
「大学は卒業したが無職」も発生し得る。

## 6. 恋愛システム

### 設計思想

- 恋愛イベントは**オプトイン型**（固定で降ってこない）
- `romance_exp` と `connections` の両方を参照して成功率を動的計算
- 告白・アプローチの成功率は**最低15%・最高85%**（100%にはならない）

### 成功率計算式

```
base_rate  = 0.20
romBonus   = (romance_exp / 10) × 0.45   → 最大 +45%
conBonus   = (connections  / 10) × 0.20  → 最大 +20%
total      = clamp(base + romBonus + conBonus, 0.15, 0.85)
```

実装：`server/index.js` に `calcRomanceChance(player)` 関数を追加。  
JSON 側の選択肢に `"dynamicRandomChance": true` フラグを付与することで動的確率を適用。

### 失敗時のダメージ（手痛く設計）

```json
"randomPenaltyEffects": {
  "romance_exp": 1,
  "health": -4,
  "time": -2,
  "connections": -1
}
```

失敗しても `romance_exp +1`（次回の成功率が上がる設計）。

### 浮気イベント（has_partner = true のときのみ発生）

```
選択肢 A: 断る（誠実ルート）
  → health +1, romance_exp +1

選択肢 B: 流される（浮気ルート）
  → 直後に強制イベント発生
    ├ 70%: 即バレ
    │       has_partner = false
    │       romance_exp -3, health -5, connections -2, money -3
    └ 30%: 今はバレていない
            flagEffects: { cheating: true }
            → 以降、毎月30%の確率で「バレた」割り込みが発動
            → バレたとき: 上記と同等のダメージ
```

**設計の意図：** ゲームとしての盛り上がりを担保しつつ、  
浮気は短期的にも長期的にもコストを払う設計にする。

## 7. 一人暮らしフラグ

| タイミング | 内容 |
|---|---|
| Month 1（1年4月） | 入学時に「どこに住む？」→ `living_alone` フラグ初期確定 |
| Month 6（1年9月） | 後期スタート「生活を見直す」→ 実家→一人暮らしへの転換可 |
| Month 13（2年4月） | 「やっぱり一人暮らし始める」→ 最終チャンス |

**ライフコスト（living_alone = true のとき）：**  
3ヶ月ごと（Month 3, 6, 9...）に `money -1` を自動適用。  
毎月ではなく四半期ごとにすることで、旧システム（4季節）との負荷を揃える。

## 8. 不運バッファーシステム

### フィールド

```
player.badLuckPoints: number  // 0〜10
```

### 蓄積ルール

| イベント種別 | 変動 |
|---|---|
| 軽度ペナルティ割り込み | +1 |
| 中度ペナルティ割り込み（緊急入院・金欠） | +2 |
| 重度ペナルティ割り込み | +3 |
| ポジティブ割り込み発動後 | -1（min 0） |

### 抽選補正

| badLuckPoints | ポジティブ | ネガティブ | 中立 |
|---|---|---|---|
| 0〜2 | 40% | 30% | 30% |
| 3〜4 | 60% | 10% | 30% |
| 5以上 | 80% | 0% | 20% |

## 9. 春休み・夏休みプール

### 春休みプール（Month 11・12・23・24・35）

プレイヤーのフラグで利用可能なルートを絞り込み、その中から選択する。

| ルート | 条件 | 主な効果 |
|---|---|---|
| A. 資格取得 | 常時 | credits+, intellect+ |
| B. 海外旅行 | money >= 4 | action_power++, money-- |
| C. 帰省・リフレッシュ | 常時 | health++, money+ |
| D. 運転免許合宿 | `has_license = false` | has_license 確定, money-- |
| E. 恋人と旅行 | `has_partner = true` | romance_exp++, money-- |
| F. バイト漬け | `job_type != null` | money++, health- |
| G. インターン参加 | 2年生以降 | work_tolerance++, credits+ |

### 夏休みプール（Month 5・17・29）

春休みプールと共通の候補を使いつつ、夏限定ルートを追加：

| ルート（夏限定） | 条件 | 主な効果 |
|---|---|---|
| H. 海・フェス | 常時 | health+, romance_exp+, money-- |
| I. サークル合宿 | `club_type != null and none` | connections+, health- |
| J. サマーインターン | 2年生以降 | work_tolerance++, credits+（任意） |
| K. OB/OG訪問 | 3年生以降 | connections++, work_tolerance+ |
